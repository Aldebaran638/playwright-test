下面是你这套测试文件的**统一注释规范**。

## 1. 必须写哪些注释

### 文件级注释必须写

每一个测试文件，在 `class` 上方**必须**写一段文件级注释。

这段注释**必须**说明：

1. 本测试文件测试的对象是什么
2. 本测试文件的整体测试思路是什么
3. 本测试文件覆盖了哪些测试方案
4. 每一种测试方案各自验证什么行为

---

### 测试函数级注释必须写

每一个测试函数上方**必须**写一行简要注释。

这行注释**必须**说明：

1. 这个测试验证的是哪一种场景
2. 这个场景下预期行为是什么

---

### 实现细节注释通常不必须写

在测试函数内部：

* 对临时计数器
* 对临时假函数
* 对中间变量
* 对 `assert`

这些位置**必须默认不写**解释性注释，除非不写就无法看懂测试意图。
测试代码内部**一定**以“测试名 + 测试函数上方注释”承担主要说明职责。

---

## 2. 哪一部分注释格式必须是什么样子的

### 文件级注释格式必须固定

文件级注释**必须**放在测试类定义上方，格式**必须**如下：

```python
# 本测试文件用于验证 xxx 的核心行为是否符合预期。
# 整体测试思路是：......
#
# 具体测试方案包括：
# 1. xxx：验证......
# 2. xxx：验证......
# 3. xxx：验证......
class TestXxx(...):
```

这里的格式要求是：

* **第一行必须**写“本测试文件用于验证……”
* **第二行必须**写“整体测试思路是：……”
* **中间必须**空一行
* **后面必须**写“具体测试方案包括：”
* **测试方案必须**用阿拉伯数字编号
* **每一条方案必须**是“一句话说明一个场景验证什么”

---

### 测试函数级注释格式必须固定

每个测试函数上方的注释**必须**采用下面格式：

```python
# 测试......时，是否......
async def test_xxx(self) -> None:
```

或者：

```python
# 测试......情况下，是否......
def test_xxx(self) -> None:
```

这里的格式要求是：

* **必须**只有一行
* **必须**以“测试”开头
* **必须**明确写出“场景 + 预期行为”
* **必须**紧贴测试函数定义，不能隔空行

---

### 测试函数命名格式必须固定

测试函数名**必须**采用行为描述式命名，格式**必须**接近下面这种：

```python
test_returns_success_result_when_step_succeeds
test_returns_failed_result_when_non_critical_step_fails
test_raises_error_when_critical_step_fails
```

要求是：

* **必须**以 `test_` 开头
* **必须**体现“什么条件下，发生什么结果”
* **必须**直接表达行为，不允许写成模糊名字，比如：

  * `test_case_1`
  * `test_run_step`
  * `test_error`
  * `test_ok`

---

## 3. 哪一部分注释必须怎么写

### 文件级第一句必须这样写

文件级注释的第一句**必须**写成：

```python
# 本测试文件用于验证 xxx 的核心行为是否符合预期。
```

这里的“xxx”**必须**是被测对象名，例如：

* `run_step_async`
* `load_json_file`
* `build_report_rows`

不能写成空泛句子，比如：

* `# 这是测试文件`
* `# 用来测试代码`
* `# 这里是单元测试`

---

### 文件级第二句必须这样写

第二句**必须**写成：

```python
# 整体测试思路是：......
```

这句话里**必须**说明测试是围绕什么行为展开的，例如：

* 成功、失败、重试
* 输入、输出、异常
* 正常场景、边界场景、错误场景

这句话**一定**不能写成“测试一下功能对不对”这种空话。

---

### 文件级“测试方案列表”必须这样写

文件级注释中的测试方案部分**必须**逐条列出，每条**必须**满足下面格式：

```python
# 1. 成功场景：验证......
# 2. 非关键失败场景：验证......
# 3. 关键失败场景：验证......
```

要求是：

* **必须**编号
* **必须**先写场景名
* **必须**再写“验证……”
* **必须**一条只说一个方案
* **必须**覆盖当前文件里真实存在的测试，不允许写文件中没有实现的方案

---

### 测试函数注释必须这样写

测试函数上方那一行注释**必须**直接对应这个测试本身，格式**必须**是：

```python
# 测试xxx情况下，是否yyy
```

例如：

```python
# 测试成功情况下，run_step_async 是否能正确返回成功结果
# 测试非关键步骤失败时，是否返回失败结果而不是抛出异常
# 测试关键步骤失败时，是否会向外抛出原始异常
```

要求是：

* **必须**写清楚测试场景
* **必须**写清楚预期行为
* **必须**与测试函数内容完全一致
* **必须**避免空泛表述，例如：

  * `# 测试是否正常`
  * `# 测试异常`
  * `# 测试重试`
    这些都不合格
